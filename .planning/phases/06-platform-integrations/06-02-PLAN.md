# Plan 06-02: YouTube Integration Service

## Goal
Create a YouTube Data API v3 service that resolves channel/playlist URLs, fetches video listings, and maps them to feed items.

## Tasks

### 1. YouTube Service
File: `src/services/youtube.ts` (new)

Functions:
- `parseYouTubeUrl(url: string)` — Extract channel ID, handle, username, or playlist ID from various YouTube URL formats
- `resolveChannelId(apiKey: string, input: { channelId?, handle?, username? })` — Resolve any identifier to a channel ID + uploads playlist ID via YouTube API
- `fetchPlaylistItems(apiKey: string, playlistId: string, maxResults?: number)` — Fetch videos from a playlist (uses playlistItems.list, 1 unit per call)
- `youtubeToFeedItems(videos: YouTubeVideo[])` — Map YouTube API response to ExtractedItem[]

Key design:
- Use `playlistItems.list` (1 unit) instead of `search.list` (100 units)
- Every channel has a hidden "Uploads" playlist: replace "UC" prefix with "UU"
- No npm dependency needed — direct `fetch()` calls to YouTube API v3

### 2. YouTube Preview Support
File: `src/routes/api/preview.ts`

Update POST /api/preview to detect YouTube URLs:
- If URL matches YouTube patterns, use YouTube service instead of web scraping
- Require YouTube API key in settings (return error if not configured)
- Return items + metadata (channel title, subscriber count)
- Set `usedHeadless: false` (no browser needed)

### 3. YouTube Feed Creation Support
File: `src/routes/api/feeds.ts`

Update POST /api/feeds:
- If URL is YouTube, set `feed_type: 'youtube'`
- Store `platform_config: { channelId, playlistId, channelTitle }`
- Store `selectors: '{}'` (not used for YouTube feeds)
- Items created from API response

### 4. YouTube Refresh Support
File: `src/routes/api/feeds.ts`

Update POST /api/feeds/:id/refresh:
- If `feed_type === 'youtube'`, use YouTube service instead of web scraping
- Fetch API key from settings table
- Fetch new videos from playlist
- Deduplicate by GUID as usual

## Verification
- [ ] `parseYouTubeUrl` handles channel, handle, playlist, and legacy URLs
- [ ] `resolveChannelId` returns channel ID + uploads playlist ID
- [ ] `fetchPlaylistItems` returns video list
- [ ] Preview API returns YouTube videos for a channel URL
- [ ] Feed creation stores feed_type='youtube' with platform_config
- [ ] Manual refresh fetches new videos via API
- [ ] Clear error when YouTube API key not configured
