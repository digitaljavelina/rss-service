# Plan 06-04: UI Updates for Platform Feeds

## Goal
Update the Create Feed page with platform type selection, build the Settings page for API key management, and add platform indicators to the dashboard.

## Tasks

### 1. Create Feed Page — Platform Tabs
File: `src/templates.ts` (update `create` page)

Add platform type selector tabs above the URL input:
- **Web** (default) — shows current form (URL + name + interval)
- **YouTube** — shows YouTube URL input + name + interval + API key notice
- **Reddit** — shows Reddit URL input + name + interval

Each tab shows/hides relevant form fields. The URL input placeholder changes per platform:
- Web: "https://news.ycombinator.com"
- YouTube: "https://youtube.com/@channelname"
- Reddit: "https://reddit.com/r/programming"

### 2. Create Feed JS — Platform-Aware Logic
File: `public/js/create-feed.js` (update)

- Track selected platform type
- When previewing YouTube: check if API key is configured (GET /api/settings/youtube_api_key), show error if missing with link to Settings
- Pass platform context to preview API (it auto-detects from URL, but UI can show platform-specific messaging)
- On save: form sends same POST /api/feeds (backend auto-detects platform from URL)

### 3. Settings Page Template
File: `src/templates.ts` (add `settings` page)

Simple settings page with:
- YouTube API Key section:
  - Password input (masked) with show/hide toggle
  - Save button
  - Test button (validates key works)
  - Help text linking to Google Cloud Console instructions
- Status indicator (configured / not configured)

### 4. Settings Page JS
File: `public/js/settings.js` (new)

- Load current settings on page load (GET /api/settings)
- Save YouTube API key (PUT /api/settings/youtube_api_key)
- Test key by calling a lightweight YouTube API endpoint
- Show success/error feedback

### 5. Settings UI Route
File: `src/routes/ui.ts` (update)

Add route: `GET /settings` → serves settings page

### 6. Dashboard — Platform Type Badges
File: `public/js/dashboard.js` (update)

- Show small badge/icon next to feed name indicating platform type
- Web feeds: no badge (default, no visual clutter)
- YouTube feeds: small "YT" badge or YouTube icon
- Reddit feeds: small "Reddit" badge

### 7. Edit Feed Page — Platform Info
File: `public/js/edit-feed.js` (update)
File: `src/templates.ts` (update `editFeed` page)

- Show feed type indicator (read-only)
- For YouTube: show channel name from platform_config
- For Reddit: show subreddit from platform_config
- Disable URL change for platform feeds (URL is derived from platform)
