  <action>
Install `pg` (^8.18.0) and `@types/pg` (^8.16.0) via `npm install pg @types/pg`.

Create `src/db/pg-adapter.ts` containing:

**PgAdapter class (singleton):**
- Constructor accepts `connectionString` from `DATABASE_URL`, creates `pg.Pool` with `max: 10`, `idleTimeoutMillis: 30_000`, `connectionTimeoutMillis: 2_000`.
- Register `pool.on('error', ...)` handler immediately (Pitfall 1 from research — unhandled pool errors crash the process).
- `getInstance()` static method returns singleton, creates Pool on first call.
- `from(table)` method returns a new `QueryBuilder` instance.
- `getPool()` method exposes the pool for health checks and migrations (needed by schema.ts in Plan 02).
- `shutdown()` async method calls `pool.end()` for graceful shutdown.

**QueryBuilder class (chainable, accumulates SQL):**

Must support exactly these 14 operation shapes found in the codebase audit (see RESEARCH.md "Complete operation inventory"):

1. `select(cols).eq(k,v).single()` — `SELECT cols FROM table WHERE k=$1 LIMIT 1`
2. `select('*').or(expr).single()` — parse Supabase `'id.eq.X,slug.eq.Y'` string format into `WHERE id=$1 OR slug=$2` (Pitfall 2)
3. `select(cols).order(col,opts).limit(n)` — `SELECT cols FROM table ORDER BY col [ASC|DESC] LIMIT n`
4. `select('*',{count:'exact',head:true}).eq(k,v)` — emit `SELECT COUNT(*)::int AS count FROM table WHERE k=$1` and return `{ data: null, count: number }` (Pitfall 3 — COUNT returns string, must parse with `parseInt` or use `::int` cast)
5. `select(cols).eq(k,v).order(col).limit(n)` — combined WHERE + ORDER + LIMIT
6. `select(cols).order(col,{ascending:false})` — ORDER BY col DESC (no WHERE)
7. `select(col).eq(k,v)` — simple WHERE query returning multiple rows
8. `insert(obj)` — single row INSERT
9. `insert(array)` — multi-row INSERT with parameterized values
10. `upsert(obj, {onConflict:col})` — INSERT ON CONFLICT DO UPDATE SET (Pitfall 7 — build SET clause from object keys excluding conflict column)
11. `update(obj).eq(k,v)` — UPDATE with WHERE
12. `update(obj).eq(k,v).select().single()` — UPDATE ... RETURNING * (detect `.select()` after `.update()` to add RETURNING)
13. `delete().eq(k,v)` — DELETE with WHERE
14. `delete().in(col,arr)` — DELETE WHERE col = ANY($1)

**Chainable method design:**
- Each method (`.select()`, `.eq()`, `.or()`, `.in()`, `.order()`, `.limit()`) stores state on the builder instance and returns `this`.
- Terminal methods (`.single()`, `.then()` / implicit await) execute the query.
- Use `then()` to make QueryBuilder thenable — allows `await supabase.from('feeds').select('*')` to resolve automatically (Supabase JS behavior).
- All queries use parameterized `$1, $2, ...` placeholders — NEVER string interpolation for values.

**Return shape:** Every operation returns `Promise<{ data: T | T[] | null; error: DbError | null; count?: number | null }>` matching the Supabase response shape.

**DbError type:** `{ code: string; message: string }` where `code` is one of: `'database_unavailable'`, `'query_failed'`, `'not_found'`, `'conflict'`, `'schema_error'`. Map pg error codes: `ECONNREFUSED`/`ETIMEDOUT` → `database_unavailable`, `23505` → `conflict`, `42P01` → `schema_error`. Default → `query_failed`. Never expose raw SQL or pg error messages to callers.

**Unsupported pattern handling (locked decision):** If a method is called that the adapter doesn't implement, log a warning via pino logger and return `{ data: null, error: null }` — do NOT throw.

**DB_DEBUG env var:** When `process.env.DB_DEBUG === 'true'`, log every SQL query string and its parameters array at debug level via pino before execution.

**Important implementation details:**
- `.or()` must parse the Supabase filter string format: split on `,`, then parse each `column.operator.value` segment. Only `eq` operator is used in the codebase.
- `.in()` must use PostgreSQL `= ANY($N)` syntax, passing the array as a single parameter.
- `.select()` with no arguments or `'*'` selects all columns. With a column list string like `'id,slug,name'`, split on comma and use those columns.
- `.order()` second argument is `{ ascending: boolean }` — default `true` if not specified.
- For `.insert()` with an array, build a multi-row VALUES clause: `($1,$2,$3),($4,$5,$6),...`.
- For `.upsert()`, the `onConflict` option specifies the conflict column. Build `ON CONFLICT (col) DO UPDATE SET` with all other columns from the object.
