  <action>
    1. Create `public/js/create-feed.js` with vanilla JavaScript (no dependencies).

    **Implementation requirements:**

    a. Get DOM element references for all form fields and sections by ID

    b. Helper function `setLoading(btn, loading)` - toggle loading spinner visibility

    c. Helper function `getFormData()` - returns object with name, url, selectors

    d. Helper function `validateForm()` - returns array of error strings
       - Check required fields: name, url, item selector, title selector
       - Validate URL format using new URL()

    e. Helper function `escapeHtml(text)` - create div, set textContent, return innerHTML
       This sanitizes any user-generated content before display

    f. Helper function `renderPreview(data)` - show preview section
       - Update previewMeta with item count
       - Clear previewItems
       - For each item (limit 10), create DOM elements using createElement
       - Use textContent for all user data (NOT innerHTML with unsanitized data)
       - Show "... and N more items" if more than 10

    g. Helper function `showPreviewError(message)` - show error in preview section

    h. btnPreview.onclick handler:
       - Validate form, show errors if any
       - Set loading state
       - POST to /api/preview with url and selectors
       - On success, call renderPreview, enable save button
       - On error, call showPreviewError
       - Clear loading state in finally

    i. btnSave.onclick handler:
       - Validate form
       - Require preview data exists
       - POST to /api/feeds with name, url, selectors
       - On success, show success section with feed URL
       - Disable form after success
       - On error, show alert

    **Security note:** Always use textContent or createElement for user data display.
    Never use innerHTML with unsanitized external content.

    2. Update `src/routes/ui.ts`:
       - Add route: uiRouter.get('/create', ...) that serves 'create' page
  </action>
