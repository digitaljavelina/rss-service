<task type="auto">
  <name>Task 2: Create extraction service layer</name>
  <files>
    src/services/page-fetcher.ts
    src/services/content-extractor.ts
    src/services/date-parser.ts
  </files>
  <action>
    1. Create `src/services/page-fetcher.ts`:
       - Export async function `fetchPage(url: string): Promise<FetchResult>`
       - Use native fetch with 10-second timeout via AbortSignal.timeout(10000)
       - Set User-Agent header: "Mozilla/5.0 (compatible; RSSService/1.0)"
       - Set Accept header: "text/html,application/xhtml+xml"
       - Return {ok: true, html} on success
       - Return {ok: false, error, statusCode} on failure
       - Wrap in try/catch to handle network errors

    2. Create `src/services/content-extractor.ts`:
       - Import cheerio and types from '../types/feed.js'
       - Export function `extractItems(html: string, baseUrl: string, selectors: FeedSelectors): ExtractedItem[]`
       - Use `cheerio.load(html)` to parse HTML
       - Query items with `$(selectors.item).slice(0, 50)` (limit to 50 items max)
       - For each item, extract title, link, description, dateText using relative selectors
       - Convert relative URLs to absolute using `new URL(href, baseUrl).href`
       - Skip items with no title AND no link
       - Return array of ExtractedItem objects

    3. Create `src/services/date-parser.ts`:
       - Import chrono-node (v2 syntax): `import * as chrono from 'chrono-node'`
       - Export function `parseDate(dateText: string | undefined, referenceDate?: Date): Date`
       - If dateText is falsy, return new Date()
       - Use `chrono.parseDate(dateText, referenceDate || new Date())`
       - If parsing fails (returns null), return new Date()
       - Return the parsed Date

    Note: All files use .js extension in imports (ES modules with TypeScript).
  </action>
