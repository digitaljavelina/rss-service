<task type="auto">
  <name>Task 1: Create Dockerfile and .dockerignore</name>
  <files>Dockerfile, .dockerignore</files>
  <action>
Create `Dockerfile` at project root with two stages:

**Stage 1: builder**
- Base: `node:22-bookworm-slim`
- WORKDIR `/app`
- Copy `package*.json`, run `npm ci` (installs all deps including devDeps for tsc)
- Copy all source files
- Run `npm run build` (builds CSS + compiles TypeScript)

**Stage 2: production**
- Base: `node:22-bookworm-slim`
- Install system Chromium and required font/lib packages via apt-get:
  `chromium`, `fonts-liberation`, `fonts-noto-cjk`, `libasound2`, `libatk1.0-0`, `libatk-bridge2.0-0`, `libcups2`, `libdbus-1-3`, `libdrm2`, `libfontconfig1`, `libgbm1`, `libgtk-3-0`, `libnspr4`, `libnss3`, `libx11-6`, `libxcomposite1`, `libxdamage1`, `libxext6`, `libxfixes3`, `libxrandr2`, `libxss1`, `libxtst6`, `xdg-utils`
- Use `--no-install-recommends` and `rm -rf /var/lib/apt/lists/*` after install
- WORKDIR `/app`
- Copy `package*.json`, run `npm ci --omit=dev` (production deps only)
- Copy `--from=builder /app/dist ./dist` (compiled JS)
- Copy `public ./public` (static assets: CSS, JS, images — referenced by `express.static('public')`)
- Do NOT copy `src/views/` — templates are embedded in `src/templates.ts` and compile into `dist/`
- Create non-root user (CNTR-05):
  ```
  RUN groupadd -r pptruser && useradd -rm -g pptruser -G audio,video pptruser
  USER pptruser
  ```
- Set environment variables:
  ```
  ENV NODE_ENV=production
  ENV PORT=3000
  ENV PUPPETEER_SKIP_CHROMIUM_DOWNLOAD=true
  ENV PUPPETEER_EXECUTABLE_PATH=/usr/bin/chromium
  ```
- `EXPOSE 3000`
- HEALTHCHECK (HLTH-02) — inline Node.js to avoid curl dependency:
  ```
  HEALTHCHECK --interval=30s --timeout=10s --start-period=30s --retries=3 \
    CMD node -e "const h=require('http');h.get('http://localhost:3000/health',r=>{process.exit(r.statusCode===200?0:1)}).on('error',()=>process.exit(1))"
  ```
- `CMD ["node", "dist/server.js"]`

Create `.dockerignore` at project root excluding:
- `node_modules/` (rebuilt inside container)
- `dist/` (produced in builder stage)
- `.env`, `.env.*` but NOT `.env.docker.example` (use `!.env.docker.example` negation)
- `.git/`, `.gitignore`
- `*.log`, `.DS_Store`
- `.planning/` (planning docs)
- `data/` (SQLite data dir, not needed)
- `docker/` (Phase 7 dev-only compose; production compose is at root)

Important: The `.dockerignore` must NOT exclude `public/` or `src/` — both are needed in the build context.

<task type="auto">
  <name>Task 2: Create docker-compose.yml and .env.docker.example</name>
  <files>docker-compose.yml, .env.docker.example</files>
  <action>
Create `docker-compose.yml` at project root (NOT in `docker/` — that one stays as Phase 7 dev reference). This is the production orchestration file. Include both services inline (no `include:` directive — avoids Docker Compose version dependency):

**postgres service:**
- `image: postgres:16-bookworm`
- Environment: `POSTGRES_DB=rssservice`, `POSTGRES_USER=rssuser`, `POSTGRES_PASSWORD=${POSTGRES_PASSWORD:-rsspassword}`
- Volume: `postgres_data:/var/lib/postgresql/data`
- Volume: `./docker/init-scripts:/docker-entrypoint-initdb.d:ro`
- Healthcheck: `pg_isready -U rssuser -d rssservice` (interval 5s, timeout 5s, retries 10, start_period 10s)
- `restart: unless-stopped`
- Do NOT expose port 5432 to host (internal-only in production)

**app service:**
- `build: .` (uses root Dockerfile)
- `ports: ["${APP_PORT:-3000}:3000"]`
- Environment (inline, not env_file — these are defaults):
  - `NODE_ENV=production`
  - `DATABASE_URL=postgresql://rssuser:${POSTGRES_PASSWORD:-rsspassword}@postgres:5432/rssservice`
  - `PORT=3000`
  - `BASE_URL=${BASE_URL:-http://localhost:3000}`
- `env_file:` with `path: .env.docker` and `required: false` (optional override file)
- `depends_on: postgres: condition: service_healthy`
- `shm_size: '256mb'` (HLTH-04 — Chromium /dev/shm stability)
- `restart: unless-stopped`

**volumes:** `postgres_data:` (named volume)

Create `.env.docker.example` at project root with sections:
1. **PostgreSQL**: `DATABASE_URL`, `POSTGRES_PASSWORD`
2. **Application**: `NODE_ENV`, `PORT`, `BASE_URL`
3. **Chromium** (comment-only — set in Dockerfile): `PUPPETEER_EXECUTABLE_PATH`
4. **Optional**: `YOUTUBE_API_KEY`, `CRON_SECRET`
5. **Port mapping**: `APP_PORT` (host port, maps to container 3000)

Each variable has a comment explaining its purpose and a sensible default or `CHANGE_ME` placeholder for secrets.
  </action>
  <verify>Run `docker compose config --quiet 2>&1` from project root (validates YAML syntax if Docker is available). Otherwise, verify YAML indentation manually via `cat docker-compose.yml`. Verify `.env.docker.example` contains DATABASE_URL and POSTGRES_PASSWORD.</verify>
  <done>docker-compose.yml at project root orchestrates app + postgres with healthcheck-gated startup, shm_size 256mb, no exposed postgres port. .env.docker.example documents all env vars.</done>
</task>
