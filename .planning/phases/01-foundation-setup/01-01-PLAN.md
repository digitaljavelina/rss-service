  <action>
Create a new Node.js project with TypeScript:

1. Run `npm init -y` to create package.json
2. Install production dependencies:
   ```
   npm install express better-sqlite3 tailwindcss daisyui feed alpinejs nanoid slugify get-port notyf compression pino
   ```
3. Install dev dependencies:
   ```
   npm install -D typescript @types/node @types/express @types/better-sqlite3 @types/compression tsx
   ```
4. Create tsconfig.json with:
   - target: ES2022
   - module: NodeNext
   - moduleResolution: NodeNext
   - outDir: dist
   - rootDir: src
   - strict: true
   - esModuleInterop: true
   - skipLibCheck: true
5. Add scripts to package.json:
   - "dev": "tsx watch src/server.ts"
   - "build": "tsc"
   - "start": "node dist/server.js"
6. Update .gitignore to include: node_modules/, dist/, data/*.db (but not data/.gitkeep)
7. Set "type": "module" in package.json for ES modules

Use tsx for development (fast TypeScript execution without compilation step).
  </action>

<task type="auto">
  <name>Task 2: Create SQLite database layer with schema</name>
  <files>
    - src/db/index.ts
    - src/db/schema.ts
    - data/.gitkeep
  </files>
  <action>
Create the database layer:

1. Create `data/` directory with `.gitkeep` file (empty file to track directory in git)

2. Create `src/db/index.ts`:
   - Import Database from better-sqlite3
   - Ensure data/ directory exists (use fs.mkdirSync with recursive: true)
   - Create database connection to `./data/rss-service.db`
   - Enable WAL mode: `db.pragma('journal_mode = WAL')`
   - Enable foreign keys: `db.pragma('foreign_keys = ON')`
   - Set busy timeout: `db.pragma('busy_timeout = 5000')`
   - Export the db instance

3. Create `src/db/schema.ts`:
   - Import db from ./index
   - Define and run schema with db.exec():
     - feeds table: id (TEXT PRIMARY KEY), slug (TEXT UNIQUE NOT NULL), name (TEXT NOT NULL), url (TEXT), selectors (TEXT for JSON), created_at, updated_at, item_limit (INTEGER DEFAULT 100)
     - items table: id (TEXT PRIMARY KEY), feed_id (TEXT NOT NULL), title, link, description, pub_date, guid (TEXT UNIQUE), created_at, FOREIGN KEY (feed_id) REFERENCES feeds(id) ON DELETE CASCADE
     - Indexes: idx_items_feed_id, idx_items_pub_date DESC, idx_feeds_slug
   - Export an `initializeDatabase()` function that runs db.exec(schema)

Note: Use TEXT for dates (SQLite stores dates as strings; ISO 8601 format sorts correctly). The guid UNIQUE constraint helps with deduplication (MGMT-05 in Phase 3).
  </action>
  <verify>
Create a simple test script or run:
```
npx tsx -e "import { initializeDatabase } from './src/db/schema.js'; initializeDatabase(); console.log('DB initialized');"
```
Verify `data/rss-service.db` file is created.
Check database has tables: `sqlite3 data/rss-service.db ".tables"` should show `feeds items`.
  </verify>
