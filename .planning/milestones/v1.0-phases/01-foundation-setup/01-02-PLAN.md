<task type="auto">
  <name>Task 2: Create feed serving with content negotiation and caching</name>
  <files>
    - src/routes/feeds.ts
    - src/services/feed-builder.ts
    - src/services/feed-cache.ts
  </files>
  <action>
Create the feed serving infrastructure:

1. Create `src/services/feed-cache.ts`:
   - Create a Map-based cache: `const feedCache = new Map<string, { xml: string; expires: number }>()`
   - CACHE_TTL = 5 * 60 * 1000 (5 minutes)
   - Export functions:
     - `getCachedFeed(cacheKey: string): string | null` - returns cached XML if not expired, null otherwise
     - `setCachedFeed(cacheKey: string, xml: string): void` - stores with expiry
     - `invalidateFeed(feedId: string): void` - removes all cache entries for a feed (both rss2 and atom1)

2. Create `src/services/feed-builder.ts`:
   - Import Feed from 'feed'
   - Import db from ../db/index
   - Import cache functions from ./feed-cache
   - Define types for FeedRow and ItemRow
   - Export `buildFeed(identifier: string, format: 'rss2' | 'atom1'): string | null`:
     - Check cache first: `getCachedFeed(\`${identifier}:${format}\`)`
     - If cached, return it
     - Query feed by slug OR id: `SELECT * FROM feeds WHERE slug = ? OR id = ?`
     - If no feed found, return null
     - Query items: `SELECT * FROM items WHERE feed_id = ? ORDER BY pub_date DESC LIMIT ?` (use feed.item_limit)
     - Create Feed instance with title, id, link (http://localhost:3000/feeds/{slug})
     - Add items to feed
     - Generate XML: format === 'atom1' ? feed.atom1() : feed.rss2()
     - Cache the result
     - Return XML

3. Create `src/routes/feeds.ts`:
   - Import Router from express
   - Import buildFeed from ../services/feed-builder
   - Create feedsRouter
   - GET /:identifier endpoint:
     - Use res.format() for content negotiation:
       - 'application/atom+xml': build Atom, set Content-Type
       - 'application/rss+xml': build RSS 2.0, set Content-Type
       - default: build RSS 2.0 (most compatible)
     - If buildFeed returns null, respond 404 with { error: 'Feed not found' }
     - Set Cache-Control: public, max-age=300 (5 min)
     - Set ETag header based on content hash (use crypto.createHash)
   - Export feedsRouter

Handle edge cases: empty feed (valid RSS with no items), missing feed (404), database errors (500).
  </action>
  <verify>
1. Start server: `npm run dev`
2. Insert a test feed into database:

   ```
   sqlite3 data/rss-service.db "INSERT INTO feeds (id, slug, name, url) VALUES ('test123', 'test-feed', 'Test Feed', 'https://example.com');"
   sqlite3 data/rss-service.db "INSERT INTO items (id, feed_id, title, link, description, pub_date, guid) VALUES ('item1', 'test123', 'Test Item', 'https://example.com/1', 'Description', datetime('now'), 'guid1');"
   ```
3. Test RSS: `curl -H "Accept: application/rss+xml" http://localhost:3000/feeds/test-feed`
   - Should return XML with <rss version="2.0">
4. Test Atom: `curl -H "Accept: application/atom+xml" http://localhost:3000/feeds/test-feed`
   - Should return XML with <feed xmlns="http://www.w3.org/2005/Atom">
5. Check headers: `curl -I http://localhost:3000/feeds/test-feed`
   - Content-Type: application/rss+xml
   - Cache-Control: public, max-age=300
6. Test 404: `curl http://localhost:3000/feeds/nonexistent`
   - Should return 404 with error message
  </verify>
