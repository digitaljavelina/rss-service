<task type="auto">
  <name>Task 2: Create dashboard.js with list, refresh, and delete handlers</name>
  <files>public/js/dashboard.js</files>
  <action>
Create public/js/dashboard.js following the IIFE pattern from create-feed.js:

1. DOM references:
   - feedList (tbody#feed-list)
   - deleteModal (dialog#delete-modal)
   - deleteFeedName (span#delete-feed-name)
   - confirmDeleteBtn (button#confirm-delete)

2. State:
   - let pendingDeleteId = null

3. loadFeeds() function:
   - fetch('/api/feeds') on page load
   - For each feed, call createFeedRow(feed) and append to feedList
   - If no feeds, show empty state with link to /create
   - Handle fetch errors with alert

4. createFeedRow(feed) function (follow RESEARCH.md pattern):
   - Create tr element with data-feed-id attribute
   - Name cell: link to /feeds/{slug}/edit with feed.name (textContent, not innerHTML)
   - Status cell: call getStatusBadge(feed) - badge-success if updated <24h, badge-warning if stale
   - Items cell: feed.itemCount
   - Updated cell: format feed.updatedAt as relative time or date
   - Actions cell: Refresh button (data-action="refresh"), Edit link, Delete button (data-action="delete", data-feed-name)
   - Return tr element

5. getStatusBadge(feed) function:
   - Calculate hours since feed.updatedAt
   - Return span with appropriate badge class (badge-success for <24h, badge-warning for stale)
   - Use textContent for badge text

6. formatDate(dateString) helper:
   - Return human-readable date (e.g., "2 hours ago" or "Feb 17, 2026")

7. Event delegation for actions:
   - document.addEventListener('click', ...) on buttons with data-action
   - If data-action="refresh": call refreshFeed(feedId)
   - If data-action="delete": set pendingDeleteId, set deleteFeedName.textContent, call deleteModal.showModal()

8. refreshFeed(feedId) function:
   - Disable refresh button, add loading class
   - POST to /api/feeds/{feedId}/refresh
   - On success: reload the feed list
   - On error: show error (alert or toast)
   - Re-enable button

9. Delete confirmation handler:
   - confirmDeleteBtn.onclick:
   - Disable button during request
   - DELETE to /api/feeds/{pendingDeleteId}
   - On success: remove row from DOM, close modal
   - On error: show alert, close modal
   - Re-enable button, reset pendingDeleteId

10. Call loadFeeds() on DOMContentLoaded
  </action>
