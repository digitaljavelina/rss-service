    2. Create `src/services/page-fetcher-browser.ts` implementing the lazy browser singleton pattern:
       - Import `chromium` from `@sparticuz/chromium` and `puppeteer` from `puppeteer-core`
       - Import `FetchResult` from `../types/feed.js`
       - Private `browserInstance` variable (lazy singleton)
       - `getBrowser()` async function:
         - If `browserInstance` exists and `browserInstance.connected`, return it
         - Set `chromium.setGraphicsMode = false` (no GPU in serverless)
         - For local dev: detect via `process.env.VERCEL` — if NOT on Vercel, use `puppeteer` (full) from devDependency with its bundled Chrome path instead. Do a dynamic `import('puppeteer')` and use its default `launch()` with `headless: 'shell'`. Wrap in try/catch — if puppeteer not available, throw clear error "Install puppeteer as devDependency for local headless support".
         - For Vercel: use `puppeteer.launch({ args: chromium.args, executablePath: await chromium.executablePath(), headless: 'shell' })`
         - Store in `browserInstance` and return
       - `fetchPageWithBrowser(url: string, timeoutMs = 20000): Promise<FetchResult>`:
         - Call `getBrowser()`
         - Open new page via `browser.newPage()`
         - Use try/finally pattern (always close page in finally)
         - `page.goto(url, { waitUntil: 'networkidle2', timeout: timeoutMs })`
         - Get `page.content()` as html
         - Return `{ ok: true, html }`
         - On error: return `{ ok: false, error: message }`
       - Export only `fetchPageWithBrowser`
  </action>
