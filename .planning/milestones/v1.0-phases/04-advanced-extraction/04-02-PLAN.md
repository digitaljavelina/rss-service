  <files>src/routes/api/preview.ts, src/routes/api/feeds.ts</files>
  <action>
    **preview.ts changes:**
    1. Import `fetchPageWithBrowser` from `../../services/page-fetcher-browser.js`
    2. Import `likelyNeedsJavaScript` from `../../services/page-fetcher.js`
    3. Add `useHeadless?: boolean` to PreviewRequest interface
    4. Add `suggestHeadless?: boolean` to PreviewResponse interface
    5. In the POST handler, after validating the request:
       - Read `const useHeadless = body.useHeadless === true;`
       - Use `fetchPageWithBrowser(body.url)` when `useHeadless` is true, otherwise use `fetchPage(body.url)` as before
       - After successful static fetch (when useHeadless is false), call `likelyNeedsJavaScript(result.html)` and include the result as `suggestHeadless` in the response
       - When useHeadless is true, set `suggestHeadless: false` (already using browser)
    6. Include `suggestHeadless` in all success response paths (both items found and no items)

    **feeds.ts changes:**
    1. Import `fetchPageWithBrowser` from `../../services/page-fetcher-browser.js`
    2. Add `useHeadless?: boolean` to CreateFeedRequest interface
    3. In POST / (create feed):
       - Read `const useHeadless = body.useHeadless === true;`
       - Use `fetchPageWithBrowser(body.url)` when useHeadless, else `fetchPage(body.url)`
       - When storing selectors, include `useHeadless` flag in the JSON: `JSON.stringify({ ...extraction.selectors, useHeadless })` — this way refresh knows to use browser
    4. In POST /:id/refresh:
       - After parsing selectors from feed row, check `selectors.useHeadless` (from stored JSON)
       - If `useHeadless` is truthy, use `fetchPageWithBrowser()` instead of `fetchPage()`
       - Remove `useHeadless` from selectors before passing to `autoExtractItems()` (it's not a CSS selector field)
    5. In PUT /:id (update feed):
       - When URL changes and re-fetching, read `useHeadless` from stored selectors JSON
       - If `useHeadless`, use `fetchPageWithBrowser()` for re-fetch
       - Preserve `useHeadless` flag when storing updated selectors
  </action>
