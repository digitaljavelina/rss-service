# Plan 06-01: Database Schema + Settings Infrastructure

## Goal
Add feed_type distinction to the database and create a settings system for API key storage, laying the groundwork for YouTube and Reddit platform feeds.

## Tasks

### 1. Database Migration: feeds table
Run in Supabase SQL Editor:
```sql
-- Add feed_type column (web, youtube, reddit)
ALTER TABLE feeds ADD COLUMN IF NOT EXISTS feed_type TEXT NOT NULL DEFAULT 'web';

-- Add platform_config for platform-specific metadata
ALTER TABLE feeds ADD COLUMN IF NOT EXISTS platform_config JSONB NOT NULL DEFAULT '{}';

-- Index for filtering by feed type
CREATE INDEX IF NOT EXISTS idx_feeds_feed_type ON feeds (feed_type);
```

### 2. Database Migration: settings table
Run in Supabase SQL Editor:
```sql
-- Global settings key-value store
CREATE TABLE IF NOT EXISTS settings (
  key TEXT PRIMARY KEY,
  value TEXT NOT NULL,
  updated_at TIMESTAMPTZ DEFAULT now()
);
```

### 3. Update TypeScript Types
File: `src/types/feed.ts`
- Add `FeedType` type: `'web' | 'youtube' | 'reddit'`
- Add `feed_type` and `platform_config` to `FeedRow` interface
- Add `PlatformConfig` interfaces for YouTube and Reddit
- Add `SettingsRow` interface

### 4. Settings API Endpoints
File: `src/routes/api/settings.ts` (new)
- `GET /api/settings` — List all settings (values masked for API keys)
- `GET /api/settings/:key` — Get single setting
- `PUT /api/settings/:key` — Create/update a setting
- `DELETE /api/settings/:key` — Delete a setting

### 5. Mount Settings API
File: `src/routes/index.ts`
- Import and mount settingsApiRouter at `/api/settings`

### 6. Update feed list API to include feed_type
File: `src/routes/api/feeds.ts`
- Include `feed_type` in GET /api/feeds response
- Include `feed_type` in GET /api/feeds/:id response
