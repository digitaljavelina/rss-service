<task type="auto">
  <name>Task 1: Create cron scheduler endpoint</name>
  <files>api/cron/scheduler.ts</files>
  <action>
    Create a new API endpoint for the Vercel cron job. This endpoint:

    1. Verifies the request is from Vercel cron (checks Authorization header)
    2. Queries feeds where `next_refresh_at <= now` and `refresh_status = 'idle'`
    3. Marks selected feeds as 'refreshing' to prevent concurrent processing
    4. Refreshes each feed (reuses existing refresh logic from feeds API)
    5. Updates `next_refresh_at` based on feed's `refresh_interval_minutes`
    6. Sets `refresh_status` back to 'idle' on success, 'error' on failure

    Key implementation details:
    - Limit to 5 feeds per cron run to stay within 60s timeout
    - Use Promise.allSettled for parallel processing
    - Reuse fetchPage/fetchPageWithBrowser and autoExtractItems
    - Store errors in last_refresh_error column
    - Calculate next_refresh_at = now + refresh_interval_minutes

    Security:
    - Check `Authorization: Bearer ${CRON_SECRET}` header
    - Return 401 if not authorized
    - Set CRON_SECRET in Vercel environment variables
  </action>
  <verify>
    - File exists at api/cron/scheduler.ts
    - TypeScript compiles: `npx tsc --noEmit`
    - Authorization check present
    - Query filters by next_refresh_at and refresh_status
  </verify>
  <done>
    Cron scheduler endpoint created with feed refresh logic
  </done>
</task>

<task type="auto">
  <name>Task 2: Configure Vercel cron job</name>
  <files>vercel.json</files>
  <action>
    Update vercel.json to add the cron job configuration:

    ```json
    {
      "functions": {
        "api/index.ts": {
          "memory": 1024,
          "maxDuration": 60
        },
        "api/cron/scheduler.ts": {
          "memory": 1024,
          "maxDuration": 60
        }
      },
      "crons": [
        {
          "path": "/api/cron/scheduler",
          "schedule": "* * * * *"
        }
      ]
    }
    ```

    Note: `* * * * *` runs every minute. This is allowed on Vercel Pro plan.
    For Hobby plan, minimum is daily - would need `0 0 * * *` instead.
  </action>
  <verify>
    - vercel.json has valid JSON syntax
    - crons array includes scheduler path
    - schedule is `* * * * *` (every minute)
    - Function memory and duration configured
  </verify>
  <done>
    Vercel cron job configured to run every minute
  </done>
</task>
